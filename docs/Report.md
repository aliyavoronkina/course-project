# Отчет о тестировании сервиса покупки туров

## Краткое описание

Проведено комплексное автоматизированное тестирование веб-сервиса по покупке туров.
Тестирование охватывало функциональность UI-форм оплаты, валидацию полей ввода
и проверку корректности интеграции с базой данных MySQL.

## Детализация тестирования

### Успешные тесты (6):
- Валидация кириллических символов в поле владельца
- Валидация нулевого месяца (00)
- Валидация невалидного номера карты
- Валидация специальных символов в имени владельца
- Валидация слишком длинного имени владельца
- Валидация цифр в поле владельца

### Неуспешные тесты (20):
- Все тесты с проверкой записи в БД (4 теста)
- Все тесты с проверкой отображения уведомлений (16 тестов)

## Критические дефекты

### Дефект #1: Отсутствие записи данных в базу данных

**Серьезность:** Критический  
**Статус:** Открыт  
**Окружение:** Windows 11, Docker, MySQL 8.0

**Описание:** После успешного заполнения формы оплаты и нажатия кнопки "Продолжить"
данные о транзакции не сохраняются в соответствующие таблицы базы данных.

**Влияние:** Невозможно отслеживать статусы платежей, нарушена основная бизнес-логика.

**Доказательства:**
- Таблицы `payment_entity` и `credit_request_entity` остаются пустыми после выполнения транзакций
- Allure отчеты показывают успешное прохождение UI-шагов, но падение на проверках БД

### Дефект #2: Не отображаются системные уведомления

**Серьезность:** Критический  
**Статус:** Открыт  
**Окружение:** Windows 11, Chrome 142

**Описание:** При вводе невалидных данных в форму оплаты системные уведомления
об ошибках не отображаются пользователю.

**Влияние:** Пользователь не получает обратной связи о ошибках ввода,
ухудшение пользовательского опыта.

**Доказательства:**
- Скриншоты в Allure отчетах показывают отсутствие уведомлений
- Элементы уведомлений присутствуют в DOM, но имеют статус `displayed:false`

## Общие рекомендации

1. **Высокий приоритет:** Исправить интеграцию с базой данных - обеспечить запись статусов операций
2. **Высокий приоритет:** Настроить отображение системных уведомлений для пользовательского фидбека
3. **Средний приоритет:** Добавить фронтенд-валидацию полей для улучшения UX
4. **Средний приоритет:** Реализовать обработку ошибок на стороне сервера
5. **Низкий приоритет:** Внедрить детальное логирование операций для упрощения отладки

## Интеграция с системами отчетности

- **Allure Report:** Настроена полная интеграция, генерируются детальные отчеты с скриншотами, логами и page source
- **Gradle Report:** Настроена базовая отчетность, предоставляет статистику выполнения тестов
- **CI/CD:** Готова к интеграции с системами непрерывной интеграции (Jenkins, GitLab CI, etc.)
