# Автоматизация тестирования сервиса покупки туров

## Описание проекта

Проект содержит автотесты для веб-сервиса по покупке туров. Тестирование включает:
- UI-тесты форм оплаты (дебетовая карта и кредит)
- Валидацию полей ввода
- Интеграцию с базой данных MySQL
- Генерацию Allure отчетов

## Технологический стек

- **Java 11** - язык программирования
- **JUnit 5** - фреймворк для тестирования
- **Selenide** - автоматизация браузера
- **Allure Report** - система отчетности
- **Gradle** - система сборки
- **Docker** - контейнеризация
- **MySQL 8.0** - база данных

## Предварительные требования

- Установленный **Docker Desktop**
- Установленный **Java 11** или выше
- Установленный **Git**

## Быстрый запуск

### 1. Запуск тестируемого сервиса и БД

docker-compose up -d

### 2 Проверка статуса сервисов

docker-compose ps
docker-compose logs app

### 3 Запуск автотестов с отчетом
./gradlew clean test allureReport allureServe

(или посмотреть вручную) ./gradlew allureServe  - откроется allure отчет

Gradle отчет:
Перейдите в папку build/reports/tests/test/ и откройте index.html

### Отчеты тестирования

После запуска тестов отчет откроется автоматически по адресу: http://localhost:port/

### Выявленные дефекты

### Критический баг #1: Данные не сохраняются в БД

Шаги воспроизведения:

1.Запустить сервис
docker-compose up -d

2.Запустить тест оплаты
./gradlew test --tests "DebitTest.shouldSuccessfullyPayWithApprovedCard"

3.Проверить БД
docker-compose exec mysql mysql -uapp -ppass app -e "SELECT * FROM payment_entity;"

Ожидаемый результат: Данные в таблице payment_entity

Фактический результат: Таблица пустая

### Критический баг #2: Не отображаются уведомления

Шаги воспроизведения:

1.Запустить тест с невалидными данными
./gradlew test --tests "InvalidTest.shouldShowErrorForEmptyForm"

2.Проверить скриншот в Allure отчете

Ожидаемый результат: Красное уведомление об ошибке

Фактический результат: Уведомление отсутствует
