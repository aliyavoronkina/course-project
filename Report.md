# Отчет о тестировании веб-сервиса покупки тура

## Краткое описание

Проведено автоматизированное тестирование веб-сервиса по покупке туров. Тестирование включало проверку функциональности двух форм оплаты:
- Покупка тура по дебетовой карте
- Покупка тура в кредит

Тестирование проводилось с использованием фреймворка Selenide для UI-тестов и интеграции с СУБД MySQL для проверки корректности сохранения данных.

## Количество тест-кейсов

Всего автоматизировано **24 тест-кейса**:

### По типам тестирования:
- **Функциональное тестирование**: 8 тестов
- **Валидационное тестирование**: 16 тестов

### По формам оплаты:
- **Тесты для формы "Купить"**: 12 тестов
- **Тесты для формы "Купить в кредит"**: 12 тестов

### По результатам выполнения:
- **Успешных**: 18 тестов (75%)
- **Неуспешных**: 6 тестов (25%)

## Общие рекомендации

### Критические проблемы:
1. **Некорректная обработка отклоненных карт** - при использовании карты 5555 6666 7777 8888 система в некоторых случаях показывает успешное уведомление вместо ошибки
2. **Проблемы с валидацией поля "Владелец"** - система допускает ввод кириллических символов, что противоречит требованиям

### Серьезные проблемы:
3. **Отсутствие валидации для истекшего срока действия карты** - система принимает карты с прошедшим месяцем/годом
4. **Некорректные сообщения об ошибках** - в некоторых случаях отображаются неинформативные или неправильные сообщения

### Улучшения:
5. **Добавить маску ввода для номера карты** - автоматическое форматирование 0000 0000 0000 0000
6. **Улучшить обработку ошибок от банковского сервиса** - более информативные сообщения для пользователя
7. **Добавить валидацию на стороне клиента** для всех полей формы

## Выводы

Автоматизированное тестирование выявило ряд критических и серьезных проблем в работе веб-сервиса. Рекомендуется устранить выявленные дефекты перед выводом системы в продакшен. Наиболее важными являются проблемы с обработкой отклоненных карт и валидацией данных, так как они могут привести к финансовым потерям и неудовлетворенности пользователей.

Система готова к дальнейшему тестированию после исправления выявленных дефектов. Рекомендуется также провести нагрузочное тестирование и тестирование безопасности перед промышленной эксплуатацией.
