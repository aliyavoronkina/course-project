# План автоматизации тестирования сервиса покупки туров

## Перечень автоматизируемых сценариев

### Позитивные сценарии:
- Успешная оплата дебетовой картой (1111 2222 3333 4444)
- Отклонение оплаты дебетовой картой (5555 6666 7777 8888)
- Успешная оплата в кредит approved картой
- Отклонение оплаты в кредит declined картой

### Негативные сценарии валидации полей:

**Номер карты:**
- Пустое поле
- Короткий номер (15 цифр)
- Номер с буквами
- Невалидный номер карты

**Месяц:**
- Пустое поле
- Невалидный месяц (13)
- Нулевой месяц (00)
- Одна цифра

**Год:**
- Пустое поле
- Просроченный год
- Будущий год (+6 лет)
- Одна цифра

**Владелец:**
- Пустое поле
- Кириллические символы
- Цифры в имени
- Специальные символы
- Слишком длинное имя

**CVC/CVV:**
- Пустое поле
- Короткий CVC (2 цифры)
- CVC с буквами
- Одна цифра

### Интеграционные проверки:
- Корректность записи статуса в payment_entity
- Корректность записи статуса в credit_request_entity
- Связь данных в order_entity

## Инструменты с обоснованием выбора

- **Java 11** - стандарт для enterprise приложений, стабильная LTS версия
- **JUnit 5** - современный фреймворк с поддержей параметризованных тестов
- **Selenide** - удобная обертка над Selenium с автоматическими ожиданиями
- **Allure Report** - мощная система отчетности с графиками и вложениями
- **Gradle** - быстрая система сборки с кэшированием зависимостей
- **Docker** - изоляция окружения БД и приложения, воспроизводимость
- **MySQL 8.0** - реляционная БД, используемая в продакшене

## Перечень и описание возможных рисков

1. **Нестабильность UI-тестов** - изменение структуры DOM, timing issues
2. **Проблемы с кодировкой** - отображение русских текстов в отчетах
3. **Блокировка ресурсов** - файлы заняты процессами при clean операции
4. **Зависимость от внешних сервисов** - падение тестов при недоступности SUT
5. **Изменение API БД** - несовместимость запросов при обновлении схемы
6. **Проблемы совместимости версий** - конфликты между версиями библиотек

## Оценка времени с учетом рисков

- Настройка инфраструктуры: 8 часов + 2 часа риски = **10 часов**
- Написание тестов: 16 часов + 4 часа риски = **20 часов**
- Отладка и отчетность: 8 часов + 2 часа риски = **10 часов**
- **Итого: 40 часов (5 рабочих дней)**

## План сдачи работ

- **День 1-2**: Настройка окружения, написание базовых тестов, интеграция с БД
- **День 3**: Реализация негативных сценариев, настройка параметризованных тестов
- **День 4**: Настройка отчетности Allure, отладка падающих тестов
- **День 5**: Подготовка документации, финальный отчет, ревью кода

**Критерии готовности:**
- Все тесты выполняются без технических ошибок
- Сгенерированы отчеты Allure и Gradle
- Подготовлена полная документация
- Код соответствует стандартам качества
